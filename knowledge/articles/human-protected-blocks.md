# Human-Protected Blocks

Category articles in WikiKnowledge serve as superstructure — summaries that tie leaf articles together into a coherent narrative. These summaries are generated by AI, but the *architectural vision* (the framing, the conceptual scaffolding, the pedagogical choices) is written by humans.

To support this dual-authorship model, category articles use HTML comment markers to delineate human-written and AI-written sections.

## Marker Syntax

### Human-Written Blocks

```markdown
<!-- human:start -->
This content was written by a human. AI summarization will
never modify, move, or delete anything between these markers.
<!-- human:end -->
```

### AI-Generated Blocks

```markdown
<!-- ai:start -->
This section is generated by AI and will be completely replaced
when the category is re-summarized after leaf articles change.
<!-- ai:end -->
```

## Rules

| Content Location | Treatment |
|-----------------|-----------|
| Between `<!-- human:start -->` and `<!-- human:end -->` | **Never touched** by AI. Preserved exactly as written. |
| Between `<!-- ai:start -->` and `<!-- ai:end -->` | **Replaced entirely** during AI re-summarization. |
| Outside any markers | **Treated as human-written** (safe by default). |

The "safe by default" rule is critical: if an author forgets to add markers, their content is protected. AI content must be *explicitly* marked to be eligible for replacement.

## Typical Category Article Structure

```markdown
---
id: "physics-simulation"
title: "Physics Simulation"
type: "category"
tags: ["physics", "simulation"]
categories: ["system-architecture"]
created: "2026-06-19T21:00:00Z"
modified: "2026-06-19T21:00:00Z"
---

# Physics Simulation

<!-- human:start -->
Physics simulation encompasses the computational methods used to
model physical phenomena. In this knowledge base, we organize
simulation techniques by the type of physical system being modeled,
starting with rigid body dynamics and progressing to soft body
and fluid simulation.

The key insight is that all simulation methods share a common
structure: a state representation, a time integration scheme,
and a constraint solver.
<!-- human:end -->

## Articles in This Category

<!-- ai:start -->
### [[rigid-body-dynamics|Rigid Body Dynamics]]
Covers the fundamentals of simulating solid objects...

### [[fluid-simulation|Fluid Simulation]]
Describes particle-based and grid-based approaches...
<!-- ai:end -->
```

## Nesting

Markers **cannot be nested**. The following is invalid:

```markdown
<!-- human:start -->
Some text
<!-- human:start -->  ← ERROR: nested open
More text
<!-- human:end -->
<!-- human:end -->
```

The parser will flag this as an error and refuse to process the article until the nesting is resolved.

## Visual Indicator

In the [[fastapi-backend|web UI]], human-protected blocks are displayed with a subtle left border (a colored bar) and a small "✍️ Human" badge. AI-generated blocks have a different colored bar and an "🤖 AI" badge. This makes the authorship of each section immediately visible to the reader.

## Implementation

The block detection is handled by a simple state-machine parser in the core module. It scans the markdown line by line, tracking whether the current position is inside a human block, an AI block, or unmarked territory. The [[storage-abstraction]] exposes this parsed structure so the summarization service knows exactly which sections to replace and which to preserve.
